# 🍡
鸣潮「小团快跑」活动的蒙特卡洛模拟器。

这个项目用于模拟团子比赛过程，统计不同团子的冠军率。脚本不会考虑游戏内赔率，因为赔率会随实际比赛和应援人数变化，本项目只分析规则本身带来的胜率差异。

## 当前默认结论

在默认规则下：

- 普通团子骰子为 1 / 2 / 3 等概率
- 布大王团子骰子为 1 / 2 / 3 / 4 / 5 / 6 等概率
- 初始堆叠顺序随机
- 每回合行动顺序随机
- 不考虑应援赔率

模拟结果中，拥有「好事成“双”」技能的达妮娅团子通常胜率最高。

示例输出：

## 规则建模

### 赛道

赛道共 32 格，第 32 格为终点。赛道是环形赛道，如果比赛继续进行，第 32 格之后会回到第 1 格。

特殊格子：

| 格子 | 装置 |
|---|---|
| 3 | 推进装置 |
| 6 | 时空裂隙 |
| 10 | 阻遏装置 |
| 11 | 推进装置 |
| 16 | 推进装置 |
| 20 | 时空裂隙 |
| 23 | 推进装置 |
| 28 | 阻遏装置 |

### 装置效果

- 推进装置：移动终点落在该格时，额外前进 1 格。
- 阻遏装置：移动终点落在该格时，额外后退 1 格。
- 时空裂隙：移动终点落在该格时，该格堆叠顺序随机打乱。
- 如果装置移动后又落到另一个装置上，会继续触发后续装置。

### 堆叠规则

- 同一格内可以有多个团子堆叠。
- 移动某个团子时，会带着自己和自己上方的团子一起移动。
- 移动到已有团子的格子时，移动来的团子叠到原有堆叠的最上方。
- 排名判断：位置更靠前者排名更高；同一格内，上层团子排名更高。

### 布大王团子

- 布大王团子初始在终点。
- 第 3 回合开始从终点向起点移动。
- 布大王团子骰子为 1-6。
- 布大王团子始终位于堆叠底部。
- 布大王团子行动时会触发赛道装置。
- 一整轮行动结束后，如果布大王团子没有和最后一名普通团子在同一格，则传送回终点。

## 扩展新比赛

代码现在按“组别配置 + 起始状态配置 + 通用模拟引擎”组织，后续维护时主要改两个位置：

1. 新增或更换团子：在 `tuanzi_race_sim.py` 顶部的 `GROUPS` 中添加组别、团子顺序、技能标识和说明。
2. 继承上一场位置：在 `PRESET_STATES` 中添加该组的赛后状态，写入每个格子的自下而上堆叠、每个团子的累计进度、下一次终点目标。
3. 复现旧比赛：使用 `--group` 和 `--state` 指定组别与起点，例如 `--group A --state after_upper`。

## 参数说明

### `--group`

组别。

可选：

```bash
--group A
--group B
```

默认：

```bash
--group A
```

### `--n`

模拟比赛次数。

默认：

```bash
--n 50000
```

次数越多，结果越稳定，但运行时间也越长。

### `--dice`

普通团子的骰子范围。

可选：

```bash
--dice 1-3
--dice 1-6
```

当前活动默认建议使用：

```bash
--dice 1-3
```

`1-6` 只是保留为测试用。

### `--state`

模拟起始状态。

可选：

```bash
--state fresh
--state after_upper
```

默认：

```bash
--state fresh
```

含义：

- `fresh`：从第 1 格重新开赛，沿用原始默认模拟。A / B 组都支持。
- `after_upper`：使用指定组别的上半场赛后站位，按环形赛道继续模拟下半场，目标为下一次冲过终点。目前 A / B 组都支持。

脚本仍兼容旧别名 `--state after_a_upper`，但 README 后续统一使用 `--group A --state after_upper`。

### `--start`

初始堆叠顺序。

可选：

```bash
--start random
--start skill_bottom_to_top
--start skill_top_to_bottom
```

默认：

```bash
--start random
```

由于游戏内没有明确显示初始底层到上层顺序，默认沿用一周年规则，随机初始堆叠。

该参数只影响 `--state fresh`。使用赛后站位预设时，堆叠顺序由预设状态决定。

### `--order`

每回合行动顺序。

可选：

```bash
--order random_each_round
--order fixed_initial
--order dice_sorted
```

默认：

```bash
--order random_each_round
```

含义：

- `random_each_round`：每回合行动顺序随机。
- `fixed_initial`：比赛开始时随机一次行动顺序，后续固定使用。
- `dice_sorted`：按本轮骰子点数排序，点数大者先动，同点随机。

当前规则依旧为“通过掷骰子随机决定行动顺序和前进步数”，所以沿用去年规则，默认采用 `random_each_round`。

### `--rank-stats`

输出额外名次统计。

```bash
--rank-stats
```

启用后，赛后站位预设会额外输出前 4 率和平均名次，便于判断 6 进 4 的晋级风险。

## 结果解读

输出中的百分比表示该团子在模拟比赛中获得冠军的比例。

例如：

```text
达妮娅团子: 22.96%
```

表示在本次模拟条件下，达妮娅团子大约赢下了 22.96% 的比赛。

这不是实际比赛的保证结果，只是基于规则和随机模拟得到的估计值。

## 已知不确定点

由于部分细节需要根据实际比赛画面确认，本项目默认采用以下合理假设：

1. 普通团子骰子仍然是 1-3（应该没变）。
2. `fresh` 状态下初始堆叠顺序随机。
3. 每回合行动顺序随机。
4. `fresh` 状态下团子到达第 32 格后立即结算冠军；赛后站位预设会按环形赛道继续，以下一次冲过终点结算。
5. 装置可以连续触发。
6. 时空裂隙会随机打乱落点格子的普通团子堆叠顺序。
7. 赛后站位预设只继承位置和堆叠，不继承上一场的技能内部状态。
8. B 组「精密演算」按首次 3、之后 2 / 1 循环建模。
9. B 组「炫彩时刻！」按互斥概率建模：20% 无法移动、60% 双倍移动、20% 正常移动。
10. B 组「视阈解明」会和本轮所有已投点数比较；布大王团子登场后也计入本轮点数。
