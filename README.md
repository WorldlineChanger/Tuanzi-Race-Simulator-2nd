# Tuanzi Race Simulator

鸣潮「团子快跑」活动的蒙特卡洛模拟器。

这个项目用于模拟团子比赛过程，统计不同团子的冠军率。脚本不会考虑游戏内赔率，因为赔率会随实际比赛和应援人数变化，本项目只分析规则本身带来的胜率差异。

## 当前默认结论

在默认规则下：

- 普通团子骰子为 1 / 2 / 3 等概率
- 布大王团子骰子为 1 / 2 / 3 / 4 / 5 / 6 等概率
- 初始堆叠顺序随机
- 每回合行动顺序随机
- 不考虑应援赔率

模拟结果中，拥有「好事成“双”」技能的达妮娅团子通常胜率最高。

示例输出：

```text
N=50000, dice=1-3, start=random, order=random_each_round, avg_round=8.39
达妮娅团子: 22.32%
菲比团子: 18.72%
陆·赫斯团子: 15.89%
西格莉卡团子: 15.70%
卡提希娅团子: 15.14%
绯雪团子: 12.22%
```

结果会因为随机种子和模拟次数有小幅波动。

## 规则建模

### 赛道

赛道共 32 格，第 32 格为终点。

特殊格子：

| 格子 | 装置 |
|---|---|
| 3 | 推进装置 |
| 6 | 时空裂隙 |
| 10 | 阻遏装置 |
| 11 | 推进装置 |
| 16 | 推进装置 |
| 20 | 时空裂隙 |
| 23 | 推进装置 |
| 28 | 阻遏装置 |

### 装置效果

- 推进装置：移动终点落在该格时，额外前进 1 格。
- 阻遏装置：移动终点落在该格时，额外后退 1 格。
- 时空裂隙：移动终点落在该格时，该格堆叠顺序随机打乱。
- 如果装置移动后又落到另一个装置上，会继续触发后续装置。

### 堆叠规则

- 同一格内可以有多个团子堆叠。
- 移动某个团子时，会带着自己和自己上方的团子一起移动。
- 移动到已有团子的格子时，移动来的团子叠到原有堆叠的最上方。
- 排名判断：位置更靠前者排名更高；同一格内，上层团子排名更高。

### 布大王团子

- 布大王团子初始在终点。
- 第 3 回合开始从终点向起点移动。
- 布大王团子骰子为 1-6。
- 布大王团子始终位于堆叠底部。
- 布大王团子行动时会触发赛道装置。
- 一整轮行动结束后，如果布大王团子没有和最后一名普通团子在同一格，则传送回终点。

## 团子技能

| 团子名称 | 技能名称 | 技能 |
|---|---|---|
| 陆·赫斯团子 | 来颗糖吧 | 触发推进装置时，额外前进 3 格；触发阻遏装置时，额外后退 1 格。 |
| 西格莉卡团子 | 日冕，帮帮忙！ | 每轮标记排名紧邻自己且更高的最多两个团子，被标记团子本回合移动距离 -1，但不会低于 1。 |
| 达妮娅团子 | 好事成“双” | 如果本次骰子点数和上一次相同，额外前进 2 格。 |
| 绯雪团子 | 引路白鸟 | 与布大王团子相遇后，之后每次移动额外前进 1 格。 |
| 卡提希娅团子 | 翻盘桥段 | 每场最多触发 1 次。自身移动结束后如果处于最后一名，则之后每次行动有 60% 概率额外前进 2 格。 |
| 菲比团子 | 岁主庇佑 | 每次移动有 50% 概率额外前进 1 格。 |

## 使用方法

需要 Python 3，不需要安装额外依赖。

```bash
python tuanzi_race_sim.py
```

指定模拟次数：

```bash
python tuanzi_race_sim.py --n 100000
```

指定随机种子：

```bash
python tuanzi_race_sim.py --n 100000 --seed 12345
```

## 参数说明

### `--n`

模拟比赛次数。

默认：

```bash
--n 50000
```

次数越多，结果越稳定，但运行时间也越长。

### `--dice`

普通团子的骰子范围。

可选：

```bash
--dice 1-3
--dice 1-6
```

当前活动默认建议使用：

```bash
--dice 1-3
```

`1-6` 只是保留为测试用。

### `--start`

初始堆叠顺序。

可选：

```bash
--start random
--start skill_bottom_to_top
--start skill_top_to_bottom
```

默认：

```bash
--start random
```

由于游戏内没有明确显示初始底层到上层顺序，默认使用随机初始堆叠。

### `--order`

每回合行动顺序。

可选：

```bash
--order random_each_round
--order fixed_initial
--order dice_sorted
```

默认：

```bash
--order random_each_round
```

含义：

- `random_each_round`：每回合行动顺序随机。
- `fixed_initial`：比赛开始时随机一次行动顺序，后续固定使用。
- `dice_sorted`：按本轮骰子点数排序，点数大者先动，同点随机。

当前规则写的是“通过掷骰子随机决定行动顺序和前进步数”，所以默认采用 `random_each_round`。

## 示例

默认模拟：

```bash
python tuanzi_race_sim.py --n 50000
```

测试固定初始堆叠：

```bash
python tuanzi_race_sim.py --n 50000 --start skill_bottom_to_top
```

测试另一种行动顺序假设：

```bash
python tuanzi_race_sim.py --n 50000 --order dice_sorted
```

## 结果解读

输出中的百分比表示该团子在模拟比赛中获得冠军的比例。

例如：

```text
达妮娅团子: 22.32%
```

表示在本次模拟条件下，达妮娅团子大约赢下了 22.32% 的比赛。

这不是实际比赛的保证结果，只是基于规则和随机模拟得到的估计值。

## 已知不确定点

由于部分细节需要根据实际比赛画面确认，本项目默认采用以下合理假设：

1. 普通团子骰子仍然是 1-3。
2. 初始堆叠顺序随机。
3. 每回合行动顺序随机。
4. 团子到达第 32 格后立即结算冠军。
5. 装置可以连续触发。
6. 时空裂隙会随机打乱落点格子的普通团子堆叠顺序。

如果之后实测发现游戏细节不同，可以通过参数或修改脚本中的规则函数进行调整。
